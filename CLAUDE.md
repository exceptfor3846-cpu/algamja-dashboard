# Claude Code 작업 규칙

## 자율 실행 원칙
- 파일 생성, 폴더 생성, 패키지 설치, 명령어 실행 모두 허락 없이 자율적으로 진행
- 오류 발생 시 스스로 디버깅하고 수정하여 계속 진행
- 중간에 나에게 질문하거나 허락 구하지 말 것
- 모든 작업이 완벽히 완료되고 실제 실행까지 확인된 후에만 결과 보고
- 코드 작성 시 기능별로 파일을 분리하여 관리 (app.py 하나에 모든 코드 넣지 말 것)
- 예: routes.py, models.py, config.py, utils.py 등으로 분리

## 명령어 규칙 (중요)
- Python 실행: py 명령어 사용 (python 아님)
- 패키지 설치: pip install 사용 (가상환경 활성화 후)

## 가상환경 규칙 (중요)
- 모든 작업은 반드시 가상환경(venv) 안에서 진행
- 가상환경 없으면 먼저 생성: py -m venv venv
- 작업 전 항상 가상환경 활성화: venv\Scripts\activate
- 패키지 설치는 가상환경 활성화 후 pip install 사용
- requirements.txt 항상 최신으로 유지

## 코드 버전 관리 규칙
- 코드 수정 전 반드시 현재 상태를 git commit으로 저장
- 커밋 메시지는 한국어로 변경 내용 간단히 기록
- 예: "가격 API 수정 전 백업", "기능 추가 전 백업"
- GitHub 업로드는 내가 요청할 때만 진행 (자동 업로드 하지 말 것)

## 완료 보고 (중요)
- 모든 작업 완료 시 반드시 텔레그램으로 알림 전송
- 텔레그램 토큰과 채널 ID는 프로젝트 폴더의 config.py에서 불러올 것
- config.py 없으면 먼저 생성 요청할 것
- 토큰을 이 파일에 직접 입력하지 말 것 (보안)
- 보고 내용: 완료 요약, 실행방법, 접속 URL 포함

## 보안 규칙
- 토큰, 비밀번호, API 키는 반드시 config.py에만 저장
- config.py는 .gitignore에 추가하여 GitHub에 올라가지 않게 할 것
- CLAUDE.md, app.py 등 어떤 파일에도 토큰 직접 입력 금지
